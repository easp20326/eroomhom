비전문가이신 사용자분께서 '오마이오픈코드'나 AI를 통해 이룸덴탈랩 포털을 처음부터 다시 제대로 구축하실 수 있도록, **[기능 설계도]**와 **[오류 방지 프롬프트]**를 메모장에 바로 복사해서 쓰실 수 있는 형태로 정리해 드립니다.

이 가이드는 현재 index.html에서 발생하던 로그인 오류, 데이터 증발, 사진 첨부 문제를 해결하는 데 초점을 맞췄습니다.

📂 [메모장 1] 시스템 전체 설계도 (Structure Plan)
1. 시스템 계층 구조

프론트엔드: HTML5, CSS3, JavaScript (React나 Vue 없이 순수 JS로 구현하여 복잡도 감소)

백엔드/DB: Supabase (인증, 데이터베이스, 파일 저장소 통합)

통계/유틸: Chart.js (매출 그래프), SheetJS (엑셀 내보내기)

2. 데이터베이스 테이블 설계 (Supabase 전용)

users (사용자): id, email, password, role(관리자/치과용), clinic_name

clinics (치과 리스트): id, name, phone, address, manager_name

cases (기공물 주문): id, clinic_id, patient_name, tooth_no(JSON), status, price, memo, photo_url, created_at

3. 필수 메뉴 구성

대시보드: 요약 카드(접수/제작/완료 개수), 최근 주문 리스트.

주문 관리: 신규 등록(치아 선택기 포함), 상태 변경, 검색.

거래처 관리: 치과 추가/삭제/수정.

통계: 월별 매출, 치과별 주문량 차트.

설정: 데이터 백업(JSON 다운로드), 내 정보 수정.

📂 [메모장 2] 단계별 AI 구현 프롬프트 (오류 방지용)
오류를 줄이려면 한 번에 다 만들지 말고, 아래 순서대로 AI에게 하나씩 시키세요.

1단계: 로그인 및 보안 세션 (가장 중요)
"Supabase Auth를 사용하여 로그인을 구현해줘. 다음 조건은 필수야:

로그인 성공 시 users 테이블의 role 정보를 읽어와 세션에 저장해줘.

페이지가 로드될 때마다 세션이 유효한지 확인하고, 유효하지 않으면 login.html로 강제 이동시켜줘.

로그인 실패 시 사용자가 알 수 있게 구체적인 에러 메시지를 붉은색 글씨로 보여줘."

2단계: 데이터 저장 및 동기화 로직
"치과(clinics)와 기공물(cases) 정보를 Supabase DB에 저장하는 기능을 만들어줘.

데이터 저장 전에는 반드시 입력값(치과명, 환자명 등)이 비어있는지 검사해줘.

데이터를 삭제할 때는 실수 방지를 위해 '정말 삭제하시겠습니까?'라는 확인창을 띄워줘.

DB 작업이 성공하면 화면의 리스트를 새로고침 없이 즉시 업데이트해줘."

3단계: 메모 및 사진 첨부 (Storage 활용)
"기공물 상세 페이지에 메모 작성과 사진 첨부 기능을 추가해줘.

사진 파일은 Supabase Storage의 'dental-photos' 버킷에 업로드하고, 저장된 URL주소만 DB의 photo_url 컬럼에 넣어줘.

업로드 전 이미지 미리보기를 보여주고, 용량이 너무 큰 파일은 경고를 띄워줘.

메모는 입력할 때마다 로컬 스토리지에 임시 저장되게 해서 브라우저가 꺼져도 글자가 남게 해줘."

4단계: 백업 및 복구 기능
"데이터 유실 방지를 위한 백업 기능을 '설정' 메뉴에 만들어줘.

cases 테이블의 모든 데이터를 JSON 파일로 내 컴퓨터에 다운로드하는 버튼을 만들어줘.

반대로 JSON 파일을 업로드하면 DB에 일괄 등록(Import)하는 기능도 추가해줘.

모든 테이블 데이터를 엑셀(XLSX)로 내려받는 기능도 포함해줘."

📂 [메모장 3] 비전문가를 위한 개발 체크리스트 (오류 방지 팁)
API Key 보안: Supabase URL과 Key가 소스코드에 직접 노출되어 다른 사람에게 보이지 않도록 주의하세요.

ID 기반 처리: 치과 이름을 수정할 때 이름으로 찾으면 오류가 납니다. 항상 id라는 고유 숫자를 기준으로 수정/삭제하세요.

상태값 통일: '제작중', '제작 중' 처럼 띄어쓰기가 다르면 통계가 꼬입니다. 상태값은 미리 정의된 영문(production, shipped)을 쓰고 화면에만 한글로 보여주세요.

이미지 처리: 사진이 안 나오는 문제는 대부분 권한 문제입니다. Supabase Storage의 버킷 권한을 'Public(공개)'으로 설정했는지 확인하세요.

💡 팁: 오마이오픈코드에서 작업하실 때, 한 번에 코드가 너무 길게 나오면 중간에 잘릴 수 있습니다. "레이아웃 먼저 짜줘" -> "그 다음 로그인 기능을 붙여줘" -> "그 다음 테이블을 만들어줘" 순서로 진행하는 것이 가장 안전합니다.
