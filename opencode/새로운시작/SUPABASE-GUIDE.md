## 2단계: SQL 테이블 생성

Supabase Dashboard → **SQL Editor** → 아래 SQL 전체를 붙여넣고 **Run**:

```sql
-- 이룸덴탈랩 포털 - Supabase 스키마 v2.0

-- ────────── 트리거·함수·테이블 초기화 ──────────
DROP TRIGGER IF EXISTS trg_sync_clinic_password ON clinics;
DROP FUNCTION IF EXISTS sync_clinic_password();
DROP TABLE IF EXISTS cases        CASCADE;
DROP TABLE IF EXISTS clinics      CASCADE;
DROP TABLE IF EXISTS prosthetics  CASCADE;
DROP TABLE IF EXISTS settings     CASCADE;

-- ────────── clinics ──────────
CREATE TABLE clinics (
  id              BIGINT       PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name            TEXT         NOT NULL,
  business_number TEXT         CHECK (business_number ~ '^\d{10}$'),
  phone           TEXT,
  director        TEXT,
  email           TEXT,
  password        TEXT,
  prices          JSONB        NOT NULL DEFAULT '{}',
  remakes         JSONB        NOT NULL DEFAULT '[]',
  price_adjust    INTEGER      NOT NULL DEFAULT 0,
  permissions     JSONB        NOT NULL DEFAULT '{"statistics":false,"sales":false,"archive":false}',
  created_at      TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- ────────── cases ──────────
CREATE TABLE cases (
  id               FLOAT8       PRIMARY KEY,
  date             TEXT,
  clinic           TEXT,
  patient          TEXT,
  patient_original TEXT,
  tooth            TEXT,
  type             TEXT,
  unit_type        TEXT         NOT NULL DEFAULT 'single',
  is_remake        BOOLEAN      NOT NULL DEFAULT false,
  status           TEXT         NOT NULL DEFAULT 'working'
                                CHECK (status IN ('working','design','manufacturing','ship','done')),
  ship_date        TEXT,
  memo             TEXT,
  done_viewed      BOOLEAN      NOT NULL DEFAULT false,
  photos           JSONB        NOT NULL DEFAULT '[]',
  chart            TEXT,
  created_at       TIMESTAMPTZ  NOT NULL DEFAULT now()
);

-- ────────── prosthetics ──────────
CREATE TABLE prosthetics (
  id    BIGINT   PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name  TEXT     NOT NULL UNIQUE,
  price INTEGER  NOT NULL DEFAULT 0 CHECK (price >= 0)
);

-- ────────── settings ──────────
CREATE TABLE settings (
  id    BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  key   TEXT   NOT NULL UNIQUE,
  value TEXT
);

-- ────────── 인덱스 ──────────
CREATE INDEX idx_cases_status    ON cases (status);
CREATE INDEX idx_cases_clinic    ON cases (clinic);
CREATE INDEX idx_cases_date      ON cases (date);
CREATE INDEX idx_cases_ship_date ON cases (ship_date);
CREATE INDEX idx_clinics_name    ON clinics (name);

-- ────────── 사업자번호 → 비밀번호 자동 동기화 트리거 ──────────
CREATE OR REPLACE FUNCTION sync_clinic_password()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.business_number IS NOT NULL AND length(NEW.business_number) >= 5 THEN
    NEW.password := right(NEW.business_number, 5);
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_sync_clinic_password
  BEFORE INSERT OR UPDATE OF business_number
  ON clinics
  FOR EACH ROW
  EXECUTE FUNCTION sync_clinic_password();

-- ────────── RLS 설정 ──────────
ALTER TABLE cases        ENABLE ROW LEVEL SECURITY;
ALTER TABLE clinics      ENABLE ROW LEVEL SECURITY;
ALTER TABLE prosthetics  DISABLE ROW LEVEL SECURITY;
ALTER TABLE settings     DISABLE ROW LEVEL SECURITY;

CREATE POLICY "cases_select" ON cases FOR SELECT TO anon USING (true);
CREATE POLICY "cases_insert" ON cases FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY "cases_update" ON cases FOR UPDATE TO anon USING (true);
CREATE POLICY "cases_delete" ON cases FOR DELETE TO anon USING (true);

CREATE POLICY "clinics_select" ON clinics FOR SELECT TO anon USING (true);
CREATE POLICY "clinics_insert" ON clinics FOR INSERT TO anon WITH CHECK (true);
CREATE POLICY "clinics_update" ON clinics FOR UPDATE TO anon USING (true);

CREATE POLICY "Allow all for prosthetics" ON prosthetics FOR ALL TO anon USING (true);
CREATE POLICY "Allow all for settings" ON settings FOR ALL TO anon USING (true);
```

## 3단계: Storage 버킷 생성

1. Supabase Dashboard → **Storage** → **New bucket**
2. **Bucket name**: `iroom-files`
3. **Public bucket**: ✅ 체크
4. **Create bucket**

Storage RLS 정책 (SQL Editor에서 실행):
```sql
CREATE POLICY "Allow public read"  ON storage.objects FOR SELECT  TO anon USING (bucket_id = 'iroom-files');
CREATE POLICY "Allow public upload" ON storage.objects FOR INSERT TO anon WITH CHECK (bucket_id = 'iroom-files');
CREATE POLICY "Allow public delete" ON storage.objects FOR DELETE TO anon USING (bucket_id = 'iroom-files');
```

## 테이블 구조 요약

| 테이블 | 주요 컬럼 |
|--------|-----------|
| `cases` | id, date, clinic, patient, tooth, type, status, ship_date, memo, photos, chart, done_viewed |
| `clinics` | id, name, business_number, phone, director, password, prices, permissions |
| `prosthetics` | id, name, price |
| `settings` | key, value |

**동기화 트리거**: 거래처에 사업자번호 입력 시 비밀번호 자동 생성 (뒷5자리)

## 문제 해결

| 증상 | 해결책 |
|------|--------|
| `406 Not Acceptable` 에러 | Supabase URL/Key 값이 정확한지 확인 |
| 파일 업로드 안 됨 | Storage 버킷 RLS 정책 확인 |
| 테이블 없음 에러 | SQL 스키마를 Supabase SQL Editor에서 실행했는지 확인 |
| CORS 에러 | Supabase Dashboard → Settings → API → CORS 확인 |
